# Release V5.1

## 版本定位

本次公共发布统一命名为 **V5.1**。虽然本地最终训练路线已经迭代到 **V10**，但公开仓库不按内部实验编号逐版发布，因此继续使用 GitHub 公共版本号来表达当前对外可交付状态。

## 核心更新

- 将公开版说明从 `v5` 统一升级到 `V5.1`。
- README 重写为当前版本口径，补齐目录结构、启动方式、推理接口、阈值建议和已知限制。
- 新增 `docs/FINAL_UPDATE.md`、`docs/DEPLOYMENT.md`、`docs/MODEL_CARD.md`。
- 更新 `CHANGELOG.md` 和 `docs/PROJECT_GUIDE.md`，明确版本映射与公开仓库边界。
- 默认模式继续固定为 `base_only`，对外说明与部署文档保持一致。

## 清理与整理

- 从默认分支移除 `checkpoints/best.pth` 大权重。
- 删除旧版 `v9` 训练 / 评估脚本。
- 删除一次性的 smoke / cloud / remote 辅助脚本。
- 保留前后端主入口、V10 主线脚本、关键配置、演示图片和论文材料。

## 重要改进

- 公共仓库更轻量，更适合克隆、浏览和维护。
- 使用者可以更快理解默认模型模式、阈值建议和部署方式。
- 历史 release 和历史 tag 全部保留，不影响旧版本获取。

## 兼容性说明

- 历史 release 不做删除。
- 历史 tag 不做覆盖。
- `services/api/main.py`、`scripts/start_backend.py`、`scripts/infer_ntire.py`、`scripts/train_v10.py` 保持可用。
- 如需复现最终模型，请自行放置 `checkpoints/best.pth`。

## 使用说明

### 启动后端

```bash
python scripts/start_backend.py
```

### 启动前端

```bash
cd frontend
npm run dev
```

### 加载模型

```text
checkpoints/best.pth
```

### 默认模式与阈值

- 模式：`base_only`
- `recall-first = 0.20`
- `balanced = 0.35`
- `precision-first = 0.35`
